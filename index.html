requisitos

Lo que la solución
tenía que ser.

⚡
Rápida
Las prestaciones de salud no pueden esperar. Cada consulta a la API tiene que responder en milisegundos para no interrumpir la atención al afiliado.
🔒
Segura
Manejamos datos sensibles de personas. Cada request debe autenticarse, autorizarse y validarse antes de devolver cualquier información.
📈
Escalable
La arquitectura tiene que poder crecer. Si mañana otro organismo necesita integrarse, el sistema debe estar listo sin reescribirse desde cero.

seguridad

Tres capas de protección.

🔑
Autenticación
Cada sistema que quiere conectarse debe identificarse con credenciales válidas. Si no está registrado, no entra. Simple.
→ tokens JWT · credenciales por organismo
🛡️
Autorización
Que alguien sea quien dice ser no significa que pueda ver todo. Cada organismo tiene acceso únicamente a los datos que le corresponden.
→ permisos por endpoint · principio de mínimo privilegio
✅
Validaciones
Antes de procesar cualquier solicitud, se verifica que los datos lleguen en el formato correcto y dentro de los rangos esperados.
→ schemas de validación · sanitización de inputs
